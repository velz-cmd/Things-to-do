## Mutation lifecycle

`capture_repository_snapshot` and `start_mission` are the only Discover-owned mutations on this surface. Both require server authentication, Zod validation, durable idempotency, a database transaction for local writes, an ActionRun, an OperationalEvent, server-truth reload, and recoverable errors. Discover never marks an identity verified, changes a program policy, selects recipients, authorizes a batch, confirms a transaction, or generates a receipt.
