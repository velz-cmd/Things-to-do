# Mission Toolbox — Capital Infrastructure Primitives

**Status:** Foundation shipped — each sidebar item is a **capability**, not a page.  
**Companion:** [POSITIONING.md](./POSITIONING.md)

---

## Principle

Cursor wins because Agents, Automations, and Rules are **core capabilities**.

RESOLVE's toolbox is **capital infrastructure** — not navigation, not Settings.

```
Mission Library    → long-running economic missions (memory)
Projects           → living economic workspaces (operating systems)
Observatories      → continuous ecosystem watch (Bloomberg watchlists + AI)
Policies           → funding philosophy (Cursor Rules for capital)
Agents             → background economic workers (not chat bots)
Automations        → event-driven capital workflows
Vaults             → programmable capital pools
Knowledge          → institutional memory of each ecosystem
Developers         → APIs, SDKs, connectors (infrastructure)
```

## Signature capabilities (roadmap)

| Capability | What it is |

## API

`GET /api/mission/toolbox` — live snapshot from sensors, treasury, opportunities, ledger.

## UX rule

Toolbox **augments** Mission conversation. It never replaces it.

Clicking Observatory pulse → injects query into Mission.  
Clicking Project → opens Community DNA + "Open in Mission".  
Execution still only happens inside the conversation when user commits.
